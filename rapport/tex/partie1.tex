\section{Les systèmes autonomes}

\subsection{Présentation de l'architecture générale}

L’architecture du projet repose sur plusieurs systèmes autonomes interconnectés entre eux. Chaque groupe administre son propre AS et met en place les mécanismes nécessaires pour assurer la communication avec les autres réseaux.\\

Dans notre cas, nous sommes responsables de l’AS 10, qui joue le rôle de fournisseur d’accès. Celui-ci assure le transit entre différents réseaux, notamment l’infrastructure de notre entreprise, un réseau externe ainsi qu’un accès destiné à des clients particuliers.\\

Les routeurs utilisés sont des équipements Arista cEOS virtualisés reliés entre eux afin de reproduire le fonctionnement d’un backbone opérateur.\\

\subsection{Choix techniques}

\subsubsection{Routage interne}

Pour le routage interne de l’AS 10, nous avons choisi d’utiliser le protocole OSPF. Ce protocole permet aux routeurs d’échanger automatiquement les routes des différents sous-réseaux présents dans l’infrastructure.\\

Tous les routeurs internes de l’AS participent à la même Area OSPF. Cela permet d’avoir un routage dynamique et d’éviter de modifier manuellement les routes en cas de coupure d’un lien entre deux routeurs.\\

Les liens entre les routeurs sont configurés avec des réseaux point-à-point en /30 afin de limiter le gaspillage d’adresses IP.

\subsubsection{Routage inter-AS}

La communication avec les autres systèmes autonomes est assurée grâce au protocole BGP.

Le routeur en bordure de l’AS établit des sessions BGP avec les AS voisins afin d’échanger les routes et d’assurer l’interconnexion entre les différents réseaux présents dans le projet.

\subsection{Services mis en place}

De multiples services ont été déployés dans l’infrastructure globale afin de répondre aux exigences du projet. Ils se divisent en deux catégories, selon qu'ils sont opérés par le fournisseur d'accès ou hébergés chez le client :

\paragraph{Services fournis par l'opérateur (AS 10) :}
\begin{itemize}
    \item Un service DHCP (accès de type « box ») permettant d’attribuer automatiquement une configuration réseau aux clients particuliers
    \item Un serveur DNS global utilisé pour la résolution des noms de domaine inter-réseaux
    \item Des mécanismes de QoS mis en place sur le backbone afin de prioriser les flux réseaux VoIP, garantissant ainsi la bande passante pour les entreprises
\end{itemize}

\vspace{0.5cm}

\paragraph{Services hébergés par le client (Réseau d'Entreprise) :}
\begin{itemize}
    \item Un serveur DNS interne gérant la zone locale de l'entreprise
    \item Un serveur Web hébergeant une page de test servant de preuve de concept pour le filtrage
    \item Un serveur LDAP permettant la gestion centralisée des utilisateurs et sécurisant l'accès VPN
    \item Un serveur Asterisk (VoIP) gérant la téléphonie sur IP interne
\end{itemize}
